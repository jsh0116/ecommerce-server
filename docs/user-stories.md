## 1. 상품 탐색 및 검색

### 1.1: 상품 검색

**As a** 고객  
**I want** 키워드로 상품을 검색하고 싶다  
**So that** 원하는 상품을 빠르게 찾을 수 있다

**Acceptance Criteria:**
- [ ] 검색창에 키워드 입력 시 자동완성이 제공된다
- [ ] 검색 결과가 1초 이내에 표시된다
- [ ] 검색 결과가 없으면 "검색 결과가 없습니다" 메시지를 표시한다
- [ ] 최근 검색어가 저장되고 삭제할 수 있다
- [ ] 인기 검색어 Top 10이 표시된다

**Test Scenarios:**
```gherkin
Given 사용자가 검색창에 "청바지"를 입력하면
When 검색 버튼을 클릭하면
Then 청바지 관련 상품 목록이 표시된다
And 검색 결과 개수가 표시된다
```

---

### 1.2: 상품 필터링

**As a** 고객  
**I want** 카테고리, 브랜드, 가격, 색상, 사이즈로 필터링하고 싶다  
**So that** 원하는 조건의 상품만 볼 수 있다

**Acceptance Criteria:**
- [ ] 여러 필터를 동시에 적용할 수 있다
- [ ] 필터 적용 시 결과가 즉시 업데이트된다
- [ ] 적용된 필터를 해제할 수 있다
- [ ] 품절 상품 제외 옵션을 선택할 수 있다
- [ ] 필터 조합이 URL에 반영되어 공유 가능하다

**Test Scenarios:**
```gherkin
Given 사용자가 상품 목록 페이지에 있고
When 브랜드 "LEVI'S"와 색상 "검정"을 선택하면
Then LEVI'S의 검정색 상품만 표시된다
And URL에 filter 파라미터가 추가된다
```

---

### 1.3: 상품 상세 조회

**As a** 고객  
**I want** 상품의 상세 정보를 보고 싶다  
**So that** 구매 결정을 할 수 있다

**Acceptance Criteria:**
- [ ] 상품 이미지를 확대하여 볼 수 있다
- [ ] 여러 각도의 이미지를 슬라이드로 볼 수 있다
- [ ] 소재, 세탁 방법 등 상세 정보가 표시된다
- [ ] 사이즈 가이드를 확인할 수 있다
- [ ] 재고 현황이 실시간으로 표시된다
- [ ] 연관 상품이 추천된다

**Test Scenarios:**
```gherkin
Given 사용자가 상품 상세 페이지에 접속하면
When 페이지가 로드되면
Then 상품 이미지, 가격, 설명이 표시된다
And 사용 가능한 색상과 사이즈 옵션이 표시된다
And 현재 재고 상태가 표시된다
```

---

### 1.4: 사이즈 추천

**As a** 고객  
**I want** 내 신체 정보에 맞는 사이즈를 추천받고 싶다  
**So that** 잘못된 사이즈 구매로 인한 반품을 줄일 수 있다

**Acceptance Criteria:**
- [ ] 신체 정보를 입력하면 추천 사이즈가 표시된다
- [ ] 구매 이력이 있으면 자동으로 추천된다
- [ ] 다른 구매자의 리뷰를 기반으로 사이즈 정보가 표시된다 ("작음", "적당", "큼")
- [ ] 사이즈 프로필을 저장할 수 있다

**Test Scenarios:**
```gherkin
Given 사용자가 키 175cm, 몸무게 70kg를 입력하고
When "사이즈 추천" 버튼을 클릭하면
Then "L 사이즈를 추천합니다" 메시지가 표시된다
And 추천 근거가 표시된다
```

## 2. 장바구니 및 주문

### 2.1: 장바구니 추가

**As a** 고객  
**I want** 상품을 장바구니에 담고 싶다  
**So that** 나중에 한 번에 구매할 수 있다

**Acceptance Criteria:**
- [ ] 상품 상세 페이지에서 장바구니에 추가할 수 있다
- [ ] 색상과 사이즈를 선택해야 추가 가능하다
- [ ] 수량을 선택할 수 있다 (최대 99개)
- [ ] 재고가 부족하면 경고 메시지가 표시된다
- [ ] 이미 장바구니에 있는 상품은 수량이 증가한다
- [ ] 비로그인 사용자도 장바구니를 사용할 수 있다 (로컬 저장)

**Test Scenarios:**
```gherkin
Given 사용자가 상품 상세 페이지에서
When 사이즈 "L"을 선택하고 "장바구니 담기"를 클릭하면
Then 장바구니에 상품이 추가된다
And "장바구니에 담겼습니다" 메시지가 표시된다
And 장바구니 아이콘에 개수가 업데이트된다
```

---

### 2.2: 장바구니 관리

**As a** 고객  
**I want** 장바구니 상품을 수정하거나 삭제하고 싶다  
**So that** 구매할 상품을 정리할 수 있다

**Acceptance Criteria:**
- [ ] 수량을 변경할 수 있다
- [ ] 개별 상품을 삭제할 수 있다
- [ ] 전체 삭제할 수 있다
- [ ] 품절된 상품은 자동으로 안내된다
- [ ] 총 금액이 자동 계산된다
- [ ] 예상 배송비가 표시된다

**Test Scenarios:**
```gherkin
Given 사용자의 장바구니에 상품이 2개 있고
When 한 상품의 수량을 3개로 변경하면
Then 총 금액이 자동으로 업데이트된다
And 배송비가 재계산된다
```

---

### 2.3: 주문 생성

**As a** 고객  
**I want** 장바구니 상품을 주문하고 싶다  
**So that** 상품을 구매할 수 있다

**Acceptance Criteria:**
- [ ] 배송지를 선택하거나 새로 입력할 수 있다
- [ ] 배송 방법을 선택할 수 있다 (일반, 당일, 새벽)
- [ ] 쿠폰을 적용할 수 있다
- [ ] 포인트를 사용할 수 있다
- [ ] 배송 요청사항을 입력할 수 있다
- [ ] 최종 금액이 명확하게 표시된다
- [ ] 주문 생성 시 재고가 15분간 예약된다

**Test Scenarios:**
```gherkin
Given 사용자가 장바구니에 상품이 있고
When "주문하기"를 클릭하고 배송지를 입력하고 결제를 진행하면
Then 주문이 생성된다
And 재고가 15분간 예약된다
And 결제 대기 상태가 된다
```

---

### 2.4: 주문 취소

**As a** 고객  
**I want** 주문을 취소하고 싶다  
**So that** 잘못 주문한 상품을 취소할 수 있다

**Acceptance Criteria:**
- [ ] 발송 전까지 취소 가능하다
- [ ] 취소 사유를 선택해야 한다
- [ ] 배송 시작 후에는 취소가 불가능하고 반품 안내가 표시된다
- [ ] 취소 시 자동으로 환불 처리된다
- [ ] 쿠폰과 포인트가 복구된다

**Test Scenarios:**
```gherkin
Given 사용자가 결제를 완료한 주문이 있고
When 배송 준비중 상태에서 "주문 취소"를 클릭하면
Then 주문이 취소된다
And 환불이 자동으로 진행된다
And 사용한 쿠폰과 포인트가 복구된다
```

## 3. 결제

### 3.1: 결제 진행

**As a** 고객  
**I want** 다양한 결제 수단으로 결제하고 싶다  
**So that** 편리한 방법으로 결제할 수 있다

**Acceptance Criteria:**
- [ ] 신용카드, 체크카드로 결제할 수 있다
- [ ] 간편결제 (카카오페이, 네이버페이)를 사용할 수 있다
- [ ] 계좌이체, 가상계좌를 선택할 수 있다
- [ ] 결제 정보가 안전하게 암호화된다 (PG사 토큰)
- [ ] 결제 승인 시 재고가 실제로 차감된다
- [ ] 결제 완료 시 SMS/이메일 알림이 발송된다

**Test Scenarios:**
```gherkin
Given 사용자가 주문서 작성을 완료하고
When 신용카드 정보를 입력하고 "결제하기"를 클릭하면
Then 결제가 승인된다
And 주문 상태가 "결제 완료"로 변경된다
And 재고가 실제로 차감된다
And 주문 완료 SMS가 발송된다
```

---

### 3.2: 결제 실패 처리

**As a** 고객  
**I want** 결제 실패 시 재시도하고 싶다  
**So that** 주문을 완료할 수 있다

**Acceptance Criteria:**
- [ ] 결제 실패 사유가 명확하게 표시된다
- [ ] 최대 3회까지 재시도할 수 있다
- [ ] 다른 결제 수단을 선택할 수 있다
- [ ] 15분 내에 결제하지 않으면 주문이 자동 취소된다
- [ ] 재고 예약이 해제된다

**Test Scenarios:**
```gherkin
Given 사용자가 결제를 진행하는데
When 카드 한도가 부족하여 결제가 실패하면
Then "카드 한도가 부족합니다" 메시지가 표시된다
And "다른 결제 수단으로 재시도" 버튼이 표시된다
And 주문은 "결제 대기" 상태로 유지된다
```

## 4. 배송

### 4.1: 배송 추적

**As a** 고객  
**I want** 주문한 상품의 배송 상태를 추적하고 싶다  
**So that** 언제 도착하는지 알 수 있다

**Acceptance Criteria:**
- [ ] 송장 번호가 등록되면 알림이 발송된다
- [ ] 배송 상태를 실시간으로 확인할 수 있다
- [ ] 배송 이벤트 타임라인이 표시된다
- [ ] 예상 배송일이 표시된다
- [ ] 배송 지연 시 안내 메시지가 표시된다

**Test Scenarios:**
```gherkin
Given 사용자의 주문이 배송 시작되고
When 배송 추적 페이지에 접속하면
Then 현재 배송 상태가 표시된다
And 택배사 이름과 송장 번호가 표시된다
And 배송 이벤트 히스토리가 표시된다
```

---

### 4.2: 배송지 변경

**As a** 고객  
**I want** 발송 전에 배송지를 변경하고 싶다  
**So that** 잘못된 주소로 배송되는 것을 방지할 수 있다

**Acceptance Criteria:**
- [ ] 배송 준비중 상태에서만 변경 가능하다
- [ ] 배송 시작 후에는 변경 불가 메시지가 표시된다
- [ ] 변경 시 배송비가 재계산된다 (제주/도서산간인 경우)

**Test Scenarios:**
```gherkin
Given 사용자의 주문이 "배송 준비중" 상태이고
When 주문 상세에서 "배송지 변경"을 클릭하고 새 주소를 입력하면
Then 배송지가 변경된다
And 변경 완료 메시지가 표시된다
```

## 5. 반품 및 교환

### 5.1: 반품 신청

**As a** 고객  
**I want** 상품을 반품하고 싶다  
**So that** 마음에 들지 않는 상품을 환불받을 수 있다

**Acceptance Criteria:**
- [ ] 배송 완료 후 7일 이내에만 신청 가능하다
- [ ] 반품 사유를 선택해야 한다
- [ ] 불량인 경우 사진을 첨부해야 한다
- [ ] 환불 방법을 선택할 수 있다 (원결제수단, 계좌)
- [ ] 반품 배송비가 자동 계산된다 (단순 변심: 6,000원)
- [ ] 예상 환불 금액이 표시된다

**Test Scenarios:**
```gherkin
Given 사용자가 배송 완료된 주문이 있고
When "반품 신청"을 클릭하고 사유를 "단순 변심"으로 선택하면
Then 반품이 신청된다
And "반품 배송비 6,000원이 차감됩니다" 메시지가 표시된다
And 예상 환불 금액이 표시된다
```

---

### 5.2: 교환 신청

**As a** 고객  
**I want** 다른 사이즈로 교환하고 싶다  
**So that** 맞는 사이즈의 상품을 받을 수 있다

**Acceptance Criteria:**
- [ ] 같은 상품의 다른 색상/사이즈를 선택할 수 있다
- [ ] 교환 가능 재고가 실시간으로 표시된다
- [ ] 재고가 없으면 환불/대기 선택 가능하다
- [ ] 교환 배송비가 표시된다 (단순 변심: 6,000원)
- [ ] 교환 승인 시 새 상품이 자동 발송된다

**Test Scenarios:**
```gherkin
Given 사용자가 배송 완료된 주문(사이즈 L)이 있고
When "교환 신청"을 클릭하고 사이즈 "XL"을 선택하면
Then XL 재고가 확인된다
And 교환이 신청된다
And 교환 배송비 6,000원이 안내된다
```

---

### 5.3: 반품/교환 진행 상태 확인

**As a** 고객  
**I want** 반품/교환 진행 상태를 확인하고 싶다  
**So that** 처리 과정을 알 수 있다

**Acceptance Criteria:**
- [ ] 반품/교환 상태가 표시된다 (요청→승인→검수→완료)
- [ ] 각 단계별 진행 날짜가 표시된다
- [ ] 반품 상품 회수 송장 번호가 표시된다
- [ ] 검수 거부 시 사유가 표시된다

**Test Scenarios:**
```gherkin
Given 사용자가 반품을 신청했고
When 반품 상세 페이지에 접속하면
Then 현재 상태가 표시된다
And 진행 타임라인이 표시된다
And 다음 단계 안내가 표시된다
```

## 6. 리뷰

### 6.1: 리뷰 작성

**As a** 고객  
**I want** 구매한 상품의 리뷰를 작성하고 싶다  
**So that** 다른 사람에게 도움이 되고 포인트를 받을 수 있다

**Acceptance Criteria:**
- [ ] 구매 확정 후에만 작성 가능하다
- [ ] 평점 (1-5점)을 선택해야 한다
- [ ] 사진을 최대 10장까지 첨부할 수 있다
- [ ] 사이즈 평가를 선택할 수 있다 ("작음", "적당", "큼")
- [ ] 일반 리뷰: 500P, 포토 리뷰: 1,000P 적립된다
- [ ] 작성 완료 시 즉시 게시된다

**Test Scenarios:**
```gherkin
Given 사용자가 구매 확정한 주문이 있고
When 리뷰 작성 페이지에서 평점 5점, 내용, 사진을 추가하고 "등록"을 클릭하면
Then 리뷰가 등록된다
And 1,000 포인트가 적립된다
And "리뷰 작성 완료" 메시지가 표시된다
```

---

### 6.2: 리뷰 조회

**As a** 고객  
**I want** 상품 리뷰를 보고 싶다

- [ ] 평균 평점과 리뷰 개수가 표시된다

```gherkin
Given 사용자가 상품 상세 페이지에서
```

- [ ] 가입 완료 시 환영 쿠폰이 자동 발급된다

```gherkin
Given 사용자가 회원가입 페이지에서

And 자동 로그인된다
```

## 9. 관리자 기능

### 9.1: 상품 등록

**As an** 관리자  
**I want** 새 상품을 등록하고 싶다  
**So that** 고객에게 판매할 수 있다

**Acceptance Criteria:**
- [ ] 상품명, 카테고리, 브랜드를 입력해야 한다
- [ ] 여러 색상과 사이즈를 조합하여 SKU를 생성할 수 있다
- [ ] 각 SKU별 가격과 재고를 입력할 수 있다
- [ ] 상품 이미지를 최대 10장까지 업로드할 수 있다
- [ ] 사이즈 가이드를 입력할 수 있다
- [ ] 등록 즉시 판매 시작되거나 예약 등록할 수 있다

**Test Scenarios:**
```gherkin
Given 관리자가 상품 등록 페이지에서
When 상품 정보를 입력하고 색상 2개, 사이즈 3개를 추가하면
Then 6개의 SKU가 자동 생성된다
And 각 SKU의 재고를 입력할 수 있다
```

---

### 9.2: 주문 관리

**As an** 관리자  
**I want** 주문을 관리하고 싶다  
**So that** 출고와 배송을 진행할 수 있다

**Acceptance Criteria:**
- [ ] 주문 목록을 상태별로 필터링할 수 있다
- [ ] 주문 상세 정보를 조회할 수 있다
- [ ] 주문 상태를 변경할 수 있다
- [ ] 송장 번호를 등록할 수 있다
- [ ] 송장 번호 등록 시 고객에게 SMS가 발송된다

**Test Scenarios:**
```gherkin
Given 관리자가 주문 관리 페이지에서 "결제 완료" 주문을 조회하고
When 주문을 선택하고 송장 번호를 입력하고 "배송 시작"을 클릭하면
Then 주문 상태가 "배송중"으로 변경된다
And 고객에게 배송 시작 SMS가 발송된다
```

---

### 9.3: 재고 관리

**As an** 관리자  
**I want** 재고를 관리하고 싶다  
**So that** 오버셀링을 방지하고 적정 재고를 유지할 수 있다

**Acceptance Criteria:**
- [ ] SKU별 재고 현황을 조회할 수 있다
- [ ] 재고를 수동으로 조정할 수 있다
- [ ] 조정 사유를 입력해야 한다
- [ ] 재고 부족 알림을 받을 수 있다
- [ ] 재고 변동 히스토리를 조회할 수 있다

**Test Scenarios:**
```gherkin
Given 관리자가 재고 관리 페이지에서 특정 SKU를 선택하고
When 재고를 +50으로 조정하고 사유를 "입고 완료"로 입력하고 저장하면
Then 가용 재고가 50개 증가한다
And 재고 히스토리에 기록된다
```

### 9.4: 반품/교환 승인

**As an** 관리자  
**I want** 반품/교환을 검수하고 승인/거부하고 싶다  
**So that** 부당한 반품을 방지하고 정상 반품을 처리할 수 있다

**Acceptance Criteria:**
- [ ] 반품/교환 요청 목록을 조회할 수 있다
- [ ] 상품 사진과 사유를 확인할 수 있다
- [ ] 승인 또는 거부할 수 있다
- [ ] 거부 시 사유를 입력해야 한다
- [ ] 승인 시 자동으로 환불/교환 발송이 진행된다

**Test Scenarios:**
```gherkin
Given 관리자가 반품 관리 페이지에서 반품 요청을 조회하고
When 상품 사진과 사유를 확인하고 "승인"을 클릭하면
Then 반품이 승인된다
And 환불이 자동으로 진행된다
And 고객에게 승인 알림이 발송된다
```
